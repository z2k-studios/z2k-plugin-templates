# JSON Field Data
A Z2K Templates JSON Package is intended to deliver field data from external sources. This page steps through the two methods for specifying field data in a [[JSON Packages Overview|Z2K Templates JSON Package]].

## Method 1: Direct Insertion
Inside the JSON Package, field-value pairs can be inserted immediately at the top level. This means that field names live right along side directives. 

Any key in the JSON object that is not a recognized directive is treated as a template field name. Its value will be used to fill in the corresponding `{{field}}` in the template.

```json
{
  "cmd": "new",
  "templatePath": "Templates/Book Note.md",
  "title": "The Art of War",
  "author": "Sun Tzu",
  "rating": 5,
  "isClassic": true
}
```

Here, `title`, `author`, `rating`, and `isClassic` are field data. They fill `{{title}}`, `{{author}}`, `{{rating}}`, and `{{isClassic}}` in the template. Because this is a JSON file, `rating` arrives as a number and `isClassic` as a boolean — their types are preserved. See [[JSON Type Conversion]] for how this differs from URI sources.


## Method 2: fieldData Field
Field data can also be bundled separately from directives using the [[fieldData]] parameter, or loaded from a file in the vault. This is particularly useful when the same field data needs to be reused across multiple commands, or when the data is generated by an external tool like Apple Shortcuts. 

The `fieldData` key can be set to either a nest json object containing a list of key-value pairs for fields and their values, or it can point to an external file that contains a json object of key-value pairs for fields. 

For more information, please see the documentation page dedicated for the [[fieldData]] directive. 
### Example nested json data field
```json
{
  "cmd": "new",
  "templatePath": "Templates/Note.md",
  "prompt": "none",
  "fieldData": {
    "title": "Inline Object",
    "category": "demo",
    "priority": 3
  }
}
```


### Example json file for field data
```json
{
  "cmd": "new",
  "templatePath": "Templates/Contact Card.md",
  "prompt": "none",
  "fieldData": "Data/contacts/new-contact.json"
}
```


## Deconflicting When Both Methods are Used
When both top-level field data keys and `fieldData` are present, top-level keys take precedence. See [[fieldData]] for the full merging rules.

For simplicity and clarity, we recommend using only one of the two methods.
