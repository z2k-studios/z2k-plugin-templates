# Name for Files
Controls the terminology the plugin uses when referring to individual files throughout its interface – command names, prompts, descriptions, and settings labels all adapt to this value.

- **Default:** `note`
- **Cannot be empty**


> [!TIP] Title
> Whatever you chose, we suggest using the "singular" version of the noun (e.g. "file" and not "files"). This will ensure the commands accurately reflect what they do (i.e. work on an individual file at a time).

## Why?
Some times you just can't get away from a religious disagreement. Obsidian calls them "notes." [[The Z2K System]] calls them "cards" to honor Luhmann. Pragmatists might call them "files." OneNote converts will insist on "pages." Or maybe you want to call them something else entirely, like "noodles" - channeling FSM to resolve this religious war. Set it to whatever term fits your mental model and the plugin's UI will follow suit.

> [!NOTE] Documentation Note
> All of the documentation for the plugin refers to individual files as "files", so please note that if you change the name for files in your system, there will be small discrepancies in the interface from the documentation.
